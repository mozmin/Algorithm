### 문제 설명

<p>초 단위로 기록된 주식가격이 담긴 배열 prices가 매개변수로 주어질 때, 가격이 떨어지지 않은 기간은 몇 초인지를 return 하도록 solution 함수를 완성하세요.</p>

<h5>제한사항</h5>

<ul>
<li>prices의 각 가격은 1 이상 10,000 이하인 자연수입니다.</li>
<li>prices의 길이는 2 이상 100,000 이하입니다.</li>
</ul>

<h5>입출력 예</h5>
<table class="table">
        <thead><tr>
<th>prices</th>
<th>return</th>
</tr>
</thead>
        <tbody><tr>
<td>[1, 2, 3, 2, 3]</td>
<td>[4, 3, 1, 1, 0]</td>
</tr>
</tbody>
      </table>
<h5>입출력 예 설명</h5>

<ul>
<li>1초 시점의 ₩1은 끝까지 가격이 떨어지지 않았습니다.</li>
<li>2초 시점의 ₩2은 끝까지 가격이 떨어지지 않았습니다.</li>
<li>3초 시점의 ₩3은 1초뒤에 가격이 떨어집니다. 따라서 1초간 가격이 떨어지지 않은 것으로 봅니다.</li>
<li>4초 시점의 ₩2은 1초간 가격이 떨어지지 않았습니다.</li>
<li>5초 시점의 ₩3은 0초간 가격이 떨어지지 않았습니다.</li>
</ul>

“가격이 떨어지는 순간, 나보다 큰 이전 값들만 처리한다.”

### 삽질 기록 🧠
1. 현재 값보다 스텍에 있는 값을 peek()로만 확인하고 pop()로 꺼내지 않았다.
2. Dequq<int[]>로 하려고 했는데, 인덱스로만 해도 풀 수 있는 문제다.
3. 이중 for문 사용하여 큐 전체를 확인하려고 했다. N값이 커서 불가능
---

### 배운 점 && 보완할 점
1. 전체를 매번 순회하는 구조는 경계하자
→ 반복문 안에 또 반복문 느낌이 보이면 → 시간복잡도 먼저 계산해보기
2. for(int[] pop : prices) 같은 형태를 썼는데, 이중 for문을 꼭 써야하는지 고민해보자.
